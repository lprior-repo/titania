# xtask Assurance Compiler (assurec) PRD

## Verdict

Build v1 only as a small deterministic assurance compiler for one pilot: TenantAccess given JwtVerified.

The north star is accepted. The broad v1 is rejected. v1 must prove that assurec can stop coherent wrongness for one finite witness decision before it grows into a broader assurance platform.

## V1 Product Claim

Given JwtVerified, TenantId, UserId, and a declared tenant membership fact, assurec prevents an AI agent from landing code that can produce TenantAccess unless the finite decision table grants it, and every claim is backed by trusted oracle provenance plus fresh CI/Moon evidence.

v1 does not claim full authentication correctness, JWT verification correctness, revocation correctness, database truthfulness, distributed correctness, or session correctness.

## Frozen V1 Scope

Build behind `cargo xtask assure` and Moon targets.

| Area | V1 Scope |
|------|----------|
| Domain | TenantAccess-style witness types |
| Logic | Finite boolean/enum decision tables |
| Expressions | Tiny typed finite expression AST |
| Proof | Kani only, bounded finite decision core |
| Tests | Generated unit, proptest, trybuild |
| Static policy | AST-grep plus source/static scanner |
| Evidence | CI/Moon-produced evidence records |
| Integrity | Generated artifact drift checking |
| Trust | Assumption ledger, TCB ledger, claim ceilings |
| CI | Moon is canonical executor |

Explicitly not v1:

- Flux
- Verus
- TLA+
- Dylint
- MIR analysis
- full Red Queen engine
- manual QA engine
- macro framework
- arbitrary Rust logic
- arbitrary state machines
- full cargo-mutants requirement

These get extension slots only. Do not build them first.

## Deterministic vs Advisory Boundary

Hard deterministic gates:

- contract-lint
- oracle-provenance-check
- typed-expression-check
- path-totality-check
- path-overlap-check
- assumption-ledger-check
- TCB-ledger-check
- claim-ceiling-check
- evidence-freshness-check

Non-deterministic advisory layer:

- black-hat-reviewer
- proof-reviewer
- test-reviewer
- compiler-feasibility reviewer
- LLM adversarial contract attack
- human review

Advisory output may produce RepairPackets or proposed OracleRecords. It does not become a compiler pass until converted into typed, checkable artifacts.

Correct command direction:

```bash
cargo xtask assure contract-lint --bead vb-123
cargo xtask assure oracle-check --bead vb-123
cargo xtask assure path-check --bead vb-123
```

`contract-attack` may exist later as a review tool. It is not a truth gate.

## Oracle Provenance

Bad:

```json
{ "source": "human" }
```

Good:

```json
{
  "source_kind": "vcs_preexisting",
  "provenance": {
    "commit": "abc123",
    "present_in_merge_base": true,
    "signer": "authorized-reviewer-key-id",
    "signature_verified": true
  }
}
```

An OracleRecord is independent only if:

- it is not generated by assurec
- it exists outside generated artifact directories
- it is present in the merge base, or has verified approval/signature from an authorized reviewer
- it maps to at least one REQ/INV/PATH
- it is included in the oracle_bank_digest
- it is consumed by tests/proofs through oracle replay

Local oracle records are usable for iteration. Landing only accepts CI-trusted oracle provenance.

## Evidence Anti-Forgery

Local JSON files are forgeable. Landing accepts only:

- `ci_signed`
- `verified_attestation`

Evidence statuses:

- `local_untrusted`
- `ci_unsigned`
- `ci_signed`
- `verified_attestation`

## V1 Expression Language

No string guards.

Allowed AST:

```rust
enum Type {
    Bool,
    Enum { name: String, variants: Vec<String> },
}

enum Expr {
    True,
    False,
    Eq { var: VarId, value: EnumVariant },
    Neq { var: VarId, value: EnumVariant },
    And(Vec<Expr>),
    Or(Vec<Expr>),
    Not(Box<Expr>),
}
```

Forbidden in v1:

- arithmetic
- string comparison
- user-defined predicates
- arbitrary Rust snippets
- path priority
- arbitrary expression parsing

## Path Decision Procedure

For v1, path checking is brute-force enumeration over finite domains.

For every valuation:

- evaluate every path guard
- require exactly one matching path

Fail if:

- zero paths match
- more than one path matches
- a path matches zero valuations
- an error variant has no producing path
- a positive oracle maps to no path
- a negative oracle maps to no path

No priority ordering in v1. Priority hides overlap.

## Reference Interpreter Independence

Use separate TCB components:

```text
assure-codegen-rust
  Pure Core IR -> Rust

assure-oracle-eval
  Pure Core IR -> evaluated expected result
```

They may share typed IR data structures. They must not share lowering logic.

`assure-oracle-eval` is small TCB:

- evaluate finite expression
- find matching path
- return expected outcome

It needs golden examples, negative overlap tests, unreachable path tests, totality tests, and mutation tests over Eq/Neq/And/Or/Not.

## JwtVerified Boundary

TenantAccess proves:

Given JwtVerified, requested TenantId, and TenantMembershipRepo result, TenantAccess is granted only when tenant claim matches requested tenant and membership exists.

TenantAccess does not prove:

- JWT signature correctness
- JWT expiry correctness
- issuer/audience correctness
- clock correctness
- JWT parser correctness
- database truthfulness
- repository consistency
- revocation behavior
- session lifetime behavior

Those claims are blocked by claim ceilings unless an upstream JWT bead closes them.

Required assumption:

```json
{
  "id": "ASM_JWT_001",
  "claim": "JwtVerified is only constructible by the upstream JWT verification bead after signature, expiry, issuer, and audience checks",
  "status": "external_dependency",
  "required_for_claims": ["REQ_AUTH_FULL_001"],
  "evidence_dependency": {
    "bead": "jwt-verified",
    "required": true
  },
  "blocks_full_auth_claim": true
}
```

Do not include ExpiredToken or InvalidSignature in TenantAccess v1. If raw JWT verification is included, the pilot is AuthenticateTenantAccess and is too large for v1.

## Witness Type Policy

Witness types must deny:

- public fields
- `pub(crate)` unchecked constructors
- `new_unchecked`
- test-only constructors inside the witness module
- `Default`
- `Copy`
- `Deserialize`
- `Serialize` by default
- `From`
- `Into`
- `Deref`
- `DerefMut`
- `AsRef` to inner private state

Allowed only by explicit policy:

- Debug, redacted/manual
- Clone, only if witness is declared copyable
- PartialEq/Eq, only if needed

Serialization rule:

- wire types deserialize
- witness types do not
- wire types must pass checked constructors or verifier functions to produce witnesses

## Shell/Action Boundary

Pure core can be correct while shell code botches behavior. v1 shell code is constrained by generated boundary contracts.

Shell may:

- call TenantMembershipRepo
- classify tenant claim relation
- map repo errors to MembershipLookupFailed
- call generated decide_tenant_access

Shell must not:

- directly construct TenantAccess
- swallow repo errors into NoMembership
- treat MissingTenantClaim as TenantMismatch
- call network/time/env/fs except declared capabilities
- construct TenantAccess from anything except AccessGranted

Generated tests must exercise public API behavior, not only generated internals.

## Claim Ceilings

Claim ceilings affect pass/fail.

A requirement is closed only if:

- none of its required subjects appear in `not_proven`
- all blocking assumptions have evidence or are explicitly scoped out
- the claim ceiling is strong enough for the risk tier

TenantAccess v1 ceiling:

```json
{
  "id": "CC_TENANT_ACCESS_V1",
  "claim_ceiling": "Authorization decision correctness given JwtVerified and declared repo fact",
  "not_proven": [
    "JWT signature verification",
    "JWT expiry verification",
    "JWT parser correctness",
    "database truthfulness",
    "revocation",
    "temporal session behavior"
  ],
  "blocks_claims": [
    "REQ_AUTH_FULL_001",
    "REQ_EXPIRED_TOKEN_REJECTED",
    "REQ_INVALID_SIGNATURE_REJECTED"
  ]
}
```

## Baseline Failure Model

Each gate outcome is one of:

- satisfied
- known_failure
- new_failure
- waived

Landing rejects:

- new_failure
- known_failure in affected bead
- waiver without owner/reason/expiry
- failure signature changed from baseline

## Named Mutation Plan

Full cargo-mutants is not mandatory in v1. v1 uses named mutations per decision path first.

Example for PATH_TENANT_MISMATCH:

- mutate Err(TenantMismatch) -> Ok(AccessGranted)
- mutate Err(TenantMismatch) -> Err(NoMembership)
- delete path
- change guard DiffersFromRequested -> MatchesRequested

Full cargo-mutants belongs to nightly, release, or Tier 4 gates.

## TCB Growth Policy

Every trusted component gets a ledger entry with owner, reason, risk, introduction commit, review expiry, and required tests.

Landing fails if:

- new TCB entry lacks owner
- new TCB entry lacks reason
- new TCB entry lacks expiry/review date
- TCB grew without approval
- trusted stub was added without a ledger entry
- assumption was added without a ledger entry

## Generated Artifact Immutability

Generated artifacts are immutable and content-addressed. Drift checking covers all generated projections, not just Rust source.

CI fails if any generated source, test, AST-grep rule, Kani harness, manifest, evidence obligation, or artifact digest differs from the manifest.

## TenantAccess Pilot

Scope proves TenantAccess can be produced only when:

- tenant claim exists
- tenant claim matches requested tenant
- membership exists

Scope does not prove JWT signature, JWT expiry, parser correctness, repo truthfulness, database consistency, revocation, or temporal behavior.

Required types:

- TenantId checked newtype
- UserId checked newtype
- JwtVerified upstream witness, assumed with evidence dependency
- TenantAccess local witness
- TenantClaimRel finite enum
- MembershipFact finite enum
- AuthError exact error taxonomy

Required errors:

- MissingTenantClaim
- TenantMismatch
- MembershipLookupFailed
- NoMembership

Decision facts:

```rust
struct AccessFacts {
    user_id: UserId,
    requested_tenant: TenantId,
    tenant_claim_rel: TenantClaimRel,
    membership: MembershipFact,
}
```

Decision paths:

| Guard | Outcome |
|-------|---------|
| Missing | MissingTenantClaim |
| DiffersFromRequested | TenantMismatch |
| MatchesRequested + LookupFailed | MembershipLookupFailed |
| MatchesRequested + NotExists | NoMembership |
| MatchesRequested + Exists | Ok(AccessGranted) |

Required independent oracles:

- EX_NEG_MISSING_CLAIM
- EX_NEG_TENANT_MISMATCH
- EX_NEG_LOOKUP_FAILED
- EX_NEG_NO_MEMBERSHIP
- EX_POS_GRANTED

None may be generated.

## Implementation Milestones

1. Schema freeze: parse schemas, reject invalid examples, stabilize canonical digests
2. TenantAccess fixture: contract, trusted oracle records, claim ceiling, JwtVerified assumption
3. IR builder: contract -> typed IR, finite facts, decision paths, typed outcomes
4. Path checker: finite valuation enumeration, totality, overlap, unreachable paths, oracle mapping
5. Codegen: witness/domain/error/core/action contract generation with content-addressed artifacts
6. Static policy: AST-grep, source scanner, derive/serde checker, generated drift checker
7. Tests: unit, proptest, trybuild misuse tests with exact assertions
8. Kani: bounded decision harnesses, assumption audit, bound rationale
9. Evidence closure: trusted evidence, freshness, claim ceilings, baseline classification
10. Moon integration: trusted CI path and local evidence rejection for landing
